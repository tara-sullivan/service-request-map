## Summary notes

**Problem:** Marimo sandboxes `mo.Html()` in a restricted iframe that blocks external scripts and `fetch()` requests. This meant Leaflet.js (and any other external JS library) couldn't load, producing a blank cell.

**Solution:** Use `mo.iframe()` instead of `mo.Html()`. It uses a less restrictive iframe that allows external scripts to load. The content must be a full HTML document (with `<html>`, `<head>`, `<body>` tags) rather than an HTML fragment.

**Why Leaflet instead of Folium:** Folium is a Python wrapper around Leaflet.js. In a WASM context (and for this iframe approach), Python packages like Folium aren't available — only JavaScript runs in the browser. So we cut out the middleman and write Leaflet.js directly.

---

**Translation key for an LLM:** If you hand this to an LLM and say "convert my Folium map to work in marimo", the prompt additions to include are:

- *"Use `mo.iframe()` with a full HTML document, not `mo.Html()`"*
- *"Replace Folium with Leaflet.js loaded from a CDN"*
- *"The GeoJSON data should be serialized with `.to_json()` and injected as a Python f-string into the JavaScript"*
- *"Escape all curly braces in the JS with double braces `{{}}` since it's inside an f-string"*
